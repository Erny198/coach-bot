# 🚀 Пошаговая инструкция: деплой бота на GitHub + Railway

---

## ШАГ 1. Создаём репозиторий на GitHub

1. Откройте **github.com** → залогиньтесь (или зарегистрируйтесь)
2. Нажмите зелёную кнопку **«New»** (или перейдите на github.com/new)
3. Заполните:
   - **Repository name:** `coach-bot`
   - **Visibility:** Private (рекомендуется, т.к. будут API-ключи в настройках)
   - Поставьте галочку **«Add a README file»**
4. Нажмите **«Create repository»**

---

## ШАГ 2. Загружаем файлы в репозиторий

### Вариант А — Через браузер (самый простой):

1. Откройте ваш новый репозиторий
2. Нажмите **«Add file»** → **«Upload files»**
3. Перетащите ВСЕ файлы из скачанной папки `coach_bot`:
   - `bot.py`
   - `knowledge_base.py`
   - `requirements.txt`
   - `Procfile`
   - `runtime.txt`
   - `.gitignore`
   - `Dockerfile`
   - `README.md`
4. Нажмите **«Commit changes»**

### Вариант Б — Через терминал (если умеете с Git):

```bash
git clone https://github.com/ВАШ_ЮЗЕРНЕЙМ/coach-bot.git
cd coach-bot
# Скопируйте все файлы из coach_bot в эту папку
cp -r /путь/к/coach_bot/* .
git add .
git commit -m "Initial commit: coaching bot"
git push origin main
```

⚠️ **ВАЖНО:** НЕ загружайте файл `.env` с вашими ключами! Ключи мы добавим в Railway.

---

## ШАГ 3. Подключаем Railway

1. Откройте **railway.app**
2. Нажмите **«Login»** → **«Login with GitHub»** → разрешите доступ
3. На главной нажмите **«New Project»**
4. Выберите **«Deploy from GitHub repo»**
5. Найдите ваш репозиторий **«coach-bot»** и выберите его
6. Railway начнёт деплой (пока он упадёт — это нормально, нет ключей)

---

## ШАГ 4. Добавляем API-ключи в Railway

Это самый важный шаг — здесь мы указываем токены:

1. В Railway откройте ваш проект → кликните на сервис
2. Перейдите во вкладку **«Variables»**
3. Добавьте 3 переменные (кнопка «New Variable»):

| Variable | Value |
|----------|-------|
| `TELEGRAM_TOKEN` | Ваш токен от @BotFather |
| `ANTHROPIC_API_KEY` | Ваш ключ от Anthropic |
| `CLAUDE_MODEL` | `claude-sonnet-4-20250514` |

4. Railway автоматически перезапустит деплой

---

## ШАГ 5. Настраиваем тип сервиса

По умолчанию Railway может попытаться запустить как web-сервис. Нам нужен worker:

1. В Railway → ваш сервис → вкладка **«Settings»**
2. В секции **«Deploy»** найдите **«Start Command»**
3. Убедитесь что там: `python bot.py`
4. Если Railway спрашивает про порт/домен — это НЕ нужно, бот работает по polling

---

## ШАГ 6. Проверяем что бот работает

1. В Railway → вкладка **«Deployments»** — статус должен быть **«Active»** (зелёный)
2. Во вкладке **«Logs»** вы должны увидеть: `✅ Бот запущен!`
3. Откройте Telegram → найдите вашего бота → нажмите **«/start»**
4. Должны появиться 3 кнопки: Коуч, Коучи, Тренер

---

## 🔧 Если что-то не работает

### Бот не запускается:
- Проверьте **Logs** в Railway — там будет ошибка
- Убедитесь что все 3 переменные добавлены (Variables)
- Проверьте что `Procfile` загружен (именно с большой P)

### Бот не отвечает в Telegram:
- Проверьте что `TELEGRAM_TOKEN` правильный (скопируйте заново из BotFather)
- Убедитесь что деплой Active (зелёный) в Railway

### Claude не отвечает:
- Проверьте `ANTHROPIC_API_KEY` — скопируйте заново из console.anthropic.com
- Проверьте баланс на аккаунте Anthropic

---

## 📦 Как обновлять бота

После изменения кода:

1. Загрузите обновлённые файлы на GitHub (через браузер или `git push`)
2. Railway **автоматически** подхватит изменения и перезапустит бота

## 🔑 Где взять токены (напоминание)

### Telegram Bot Token:
1. Откройте @BotFather в Telegram
2. Отправьте `/newbot`
3. Придумайте имя и username для бота
4. Скопируйте токен (длинная строка с цифрами и буквами)

### Anthropic API Key:
1. Откройте console.anthropic.com
2. Зарегистрируйтесь / залогиньтесь
3. Settings → API Keys → Create Key
4. Скопируйте ключ (начинается с `sk-ant-...`)
